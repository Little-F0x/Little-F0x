## Reporting a Vulnerability

If you discover a security vulnerability, please follow these steps:

1. **Do not disclose the vulnerability publicly**
2. Send an email to `your_email@example.com` with:
   - A detailed description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

I will acknowledge receipt within 48 hours and provide a timeline for addressing the issue.

## Security Best Practices

### For Contributors
- Never commit sensitive data (API keys, passwords, tokens)
- Use environment variables for configuration
- Keep dependencies updated
- Follow the principle of least privilege

### For Users
- Keep the software updated to the latest version
- Review permissions granted to the application
- Monitor for unusual activity
